How I Built My Site

Technologies I use and why

August 13, 2018

The Stack

I'm registered through Google Domains ($1/month) and hosted on GitHub Pages ($0/month). I'm using Bootstrap to help control the layout and formatting but I'm not using any other libraries. This is a static site, everything is "hand-written" with Visual Studio Code and version controlled with Git. The graphics I'm using I've either created myself with GIMP or purchased from The Noun Project. My contact form is setup with formspree.

Use Case

Now I'll provide some context for the technologies I've listed above. I'm not doing anything fancy. I specifically wanted to avoid adding complexity. I set out to gain experience with the foundational front-end technologies and standards that run the modern web. There are a ton of cool tools and libraries available to speed things up, and I may choose to grow into some of those tools (Jekyll looks pretty nice) but for now I'm driven to keep it simple.

I wanted to get comfortable with how the markup (html) and stylesheets (css) interact and how to manipulate the presentation with client-side code (javascript). If you're reading this, you might be looking for the same kind of experience. Go ahead and open up the dev tools section of the browser and take a look around. Alternatively, check out the repository on GitHub.

When I started looking into building a website, I found that there are a ton of places to pick up information and examples on specific topics, but I had a hard time finding something that would show you the whole picture. You could argue that's because the majority of sites don't use static pages that are written from scratch. I'd agree with that, but for learning purposes, I think it does help immensely to look at one complete, working example.

Recommendations

I really like how everything is working together. When I started thinking about how I would host my site, I was unaware of GitHub Pages, so that kind of stole the show for me. It's part of GitHub, zero-cost, and you can have a site up in minutes. I had to read through some documentation to get my custom domain working but it's really just a matter of a couple DNS settings. A single word of warning, at this time GitHub Pages does not support any sort of back-end processing, which is totally understandable for the price. I'm more familiar with back-end technologies than front-end but I've never gotten anything hosted out on the web. I've always just ran locally through Visual Studio and IIS. Eventually I'll want to get some of that out on the web so that means I'll have to pay for hosting. The point is, if you want to jump into server-side code right away, you won't be using GitHub Pages.

I have experience with Visual Studio but decided to switch to VS Code for this project. The main reason I switched is that VS Code is just so much lighter running and I've been doing a lot of the work for this site on an older core i3 laptop. I find VS Code is a good comprimise between power and agility; it's not as heavy as Visual Studio but it's a lot more immersive than something like Notepad++. Unfortunately, on the budget laptop I'm using it will occasionally lag a little, but usually not for more than two or three seconds.
